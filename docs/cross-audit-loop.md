# Cross-Audit Loop

PACE emphasizes critique before implementation, not just review after the fact.

## Purpose

The cross-audit loop exists to challenge a proposed spec, plan, or milestone set against:

- original intent
- constraints
- risks
- scope realism
- definition of done

This helps expose ambiguity early, when it is cheaper and safer to correct.

## Public Interpretation

PACE is stronger when planning artifacts are treated as reviewable engineering outputs rather than informal setup work. The cross-audit loop makes that expectation explicit.
