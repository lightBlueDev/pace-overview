# Workflow

PACE is built around a staged progression that reduces ambiguity before execution and strengthens closure after execution.

## Core Progression

```text
Spec -> Architecture Review -> Milestones -> Atomic Tasks -> Agent Execution -> Review -> Context Preservation -> Retrospective
```

## Why The Sequence Matters

The sequence is meant to prevent downstream work from inheriting unresolved uncertainty.

Each phase serves a distinct purpose:

- specification defines intent and constraints
- architecture review challenges the initial shape
- milestones structure delivery
- atomic tasks bound execution
- review checks correctness and alignment
- context preservation makes handoff viable
- retrospective improves the method over time

PACE values explicit progression because implementation quality depends heavily on what was clarified before coding began.
