# Document Stack

PACE treats documents as operational infrastructure.

## Role Separation

The document stack separates:

- canonical truth
- dynamic project state
- verification and audit

That separation helps humans and agents orient quickly and act against explicit constraints instead of reconstructing intent from partial memory.

## Why It Matters

When document roles are blurred, several problems follow:

- status drifts away from reality
- acceptance criteria become vague
- handoffs become interpretive
- closure is easy to overclaim

PACE uses a clearer document model so the project can remain legible while work is active.
